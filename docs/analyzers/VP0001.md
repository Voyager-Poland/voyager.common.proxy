# VP0001: Collection parameter not supported on GET/DELETE

| Pole | Wartość |
|------|---------|
| **ID** | VP0001 |
| **Kategoria** | Voyager.Proxy.Usage |
| **Severity** | Error |
| **Domyślnie włączona** | Tak |

## Opis

Tablica lub kolekcja prostych typów (`int[]`, `List<string>`, `IEnumerable<Guid>`, itp.) nie może być parametrem metody GET lub DELETE w interfejsie proxy. Wartości elementów kolekcji zostaną **cicho utracone** w runtime — framework serializuje metadane typu (np. `Length`, `IsReadOnly`) zamiast elementów.

## Przykład naruszenia

```csharp
[ServiceRoute("invoice")]
public interface IInvoiceService
{
    [HttpGet("get-invoice-data")]
    Task<Result<InvoiceData>> GetInvoiceDataAsync(int[] idTickets);  // VP0001
}
```

W runtime to wygeneruje żądanie:
```
GET /invoice/get-invoice-data?Length=3&LongLength=3&IsReadOnly=False&IsFixedSize=True
```

Wartości `idTickets` (`[1, 2, 3]`) zostaną **całkowicie utracone** — brak wyjątku, brak ostrzeżenia.

## Rozwiązanie

### Opcja 1: Zmień na HttpPost (zalecane)

Code Fix automatycznie zamieni `[HttpGet]` na `[HttpPost]`:

```csharp
[ServiceRoute("invoice")]
public interface IInvoiceService
{
    [HttpPost("get-invoice-data")]
    Task<Result<InvoiceData>> GetInvoiceDataAsync(int[] idTickets);  // OK
}
```

### Opcja 2: Użyj prostych parametrów

Jeśli potrzebujesz GET, zmień sygnaturę na proste typy:

```csharp
[ServiceRoute("invoice")]
public interface IInvoiceService
{
    [HttpGet("get-invoice-data")]
    Task<Result<InvoiceData>> GetInvoiceDataAsync(int idTicket);  // OK — pojedynczy parametr
}
```

### Opcja 3: Opakuj w obiekt request

```csharp
public class InvoiceQuery { public int[] IdTickets { get; set; } }

[ServiceRoute("invoice")]
public interface IInvoiceService
{
    [HttpPost("get-invoice-data")]
    Task<Result<InvoiceData>> GetInvoiceDataAsync(InvoiceQuery query);  // OK — complex type na POST
}
```

## Wykrywane typy kolekcji

| Typ parametru | Wykrywany |
|---------------|-----------|
| `T[]` (np. `int[]`, `string[]`, `Guid[]`) | Tak |
| `List<T>` | Tak |
| `IEnumerable<T>` | Tak |
| `ICollection<T>` | Tak |
| `IList<T>` | Tak |
| `IReadOnlyList<T>` | Tak |
| `IReadOnlyCollection<T>` | Tak |
| `List<ComplexType>` | Nie — complex types nie są dotknięte |

## Wykrywane metody HTTP

Diagnostyka zgłaszana jest dla metod rozpoznanych jako GET lub DELETE:

| Źródło | Rozpoznanie |
|--------|-------------|
| `[HttpGet]`, `[HttpGet("template")]` | GET |
| `[HttpDelete]`, `[HttpDelete("template")]` | DELETE |
| `[HttpMethod(HttpMethod.Get, ...)]` | GET |
| `[HttpMethod(HttpMethod.Delete, ...)]` | DELETE |
| Nazwa metody: `Get*`, `Find*`, `List*`, `Search*` | GET (konwencja) |
| Nazwa metody: `Delete*`, `Remove*` | DELETE (konwencja) |

## Kiedy reguła nie jest zgłaszana

- Parametr jest prostym typem (`int`, `string`, `Guid`, `DateTime`, enum, itp.)
- Metoda jest POST, PUT lub PATCH
- Kolekcja zawiera typy złożone (np. `List<OrderFilter>`)
- Interfejs nie jest rozpoznany jako proxy (brak `[ServiceRoute]` i brak atrybutów `[HttpMethod]`)
- Metoda jest w klasie (nie w interfejsie)
